# ТЗ на дизайн — Веб-приложение для управления танцевальной школой

## 1. Контекст проекта

Веб-приложение для администрирования танцевальной школы. Заменяет бумажный учёт платежей, журнал посещаемости и разрозненные заметки.

**Язык интерфейса:** русский  
**Платформа:** web, mobile-first (открывается в браузере, без установки)  
**Устройства:** телефон (приоритет) + десктоп  
**Визуальный стиль:** чистый, современный, нейтральный. Без конкретного брендинга. Никаких ярких акцентных цветов — спокойная палитра (белый / светло-серый фон, тёмный текст, один сдержанный акцентный цвет на выбор дизайнера). Карточный UI. Минимум декора.

---

## 2. Роли пользователей

| Роль | Описание |
|------|----------|
| **Администратор** | Полный доступ ко всему. Управляет учениками, платежами, расписанием |
| **Педагог** | Отмечает посещаемость, видит расписание, видит свои группы |
| **Ученик / Родитель** | Видит свои начисления, историю оплат и расписание своей группы |

---

## 3. Навигация

### Мобильная версия
Нижняя панель навигации (bottom navigation bar) с иконками. Количество пунктов зависит от роли.

**Администратор:** Главная · Ученики · Платежи · Расписание · Ещё (педагоги, настройки)  
**Педагог:** Расписание · Посещаемость · Профиль  
**Ученик:** Оплата · Расписание · Профиль

### Десктопная версия
Боковая панель (sidebar) слева, постоянно видна.

---

## 4. Экраны

---

### AUTH-1 — Вход в систему

**Назначение:** единственная страница, доступная без авторизации  
**Элементы:**
- Логотип / название приложения (placeholder)
- Поле: Логин
- Поле: Пароль (скрыт, с кнопкой показать)
- Кнопка «Войти»
- Строка с ошибкой (неверный логин/пароль)

**Примечание:** Регистрация только через администратора — ссылки «Зарегистрироваться» нет.

---

### ADMIN-1 — Главная (дашборд)

**Назначение:** быстрый обзор текущего состояния школы  
**Элементы:**
- Приветствие с именем администратора
- Карточка «Должники»: число учеников с долгом + общая сумма долга, кнопка «Посмотреть»
- Карточка «Занятия сегодня»: список занятий на сегодня (группа, время, зал, педагог)
- Карточка «Последние платежи»: 5 последних зафиксированных оплат (имя ученика, сумма, позиция, время)
- Быстрые действия: кнопки «Принять оплату» и «Отметить посещаемость»

---

### ADMIN-2 — Список учеников

**Назначение:** просмотр и поиск по базе учеников  
**Элементы:**
- Строка поиска по имени
- Фильтр по группе (дропдаун)
- Фильтр: активные / архивные
- Кнопка «Добавить ученика»
- Список учеников: карточка на каждого — имя, группа(ы), статус долга (иконка или бейдж «есть долг»)
- Тап на ученика → переход в профиль

---

### ADMIN-3 — Профиль ученика

**Назначение:** полная информация по ученику, его платежи и посещаемость  
**Структура:** вкладки или секции на одной странице

**Секция «Данные»:**
- ФИО ребёнка, дата рождения
- ФИО родителя / ответственного
- Телефон
- Группа(ы)
- Откуда узнали о школе
- Кнопка «Редактировать»
- Кнопка «Деактивировать» (с подтверждением)

**Секция «Начисления»:**
- Список всех начислений: позиция, сумма, статус (оплачено / не оплачено), дата
- Статус отображается цветовым маркером: зелёный = оплачено, красный = не оплачено
- Кнопка «Добавить начисление»
- На каждой строке: кнопки «Отметить оплату», «Редактировать сумму», «Удалить»
- Кнопка «Принять оплату» (быстрый переход к флоу оплаты)

**Секция «Посещаемость»:**
- Список занятий с отметкой «был» / «не был»
- Фильтр по месяцу

---

### ADMIN-4 — Создание / редактирование ученика

**Назначение:** форма добавления нового ученика или редактирования данных  
**Поля:**
- ФИО ребёнка (обязательно)
- Дата рождения (обязательно)
- ФИО родителя / ответственного (обязательно)
- Контактный телефон (обязательно)
- Откуда узнали о школе (текстовое поле, необязательно)
- Группа(ы) — мультиселект из существующих групп
- Логин (обязательно, задаётся при создании)
- Пароль (обязательно, задаётся при создании)

**Кнопки:** «Сохранить», «Отмена»

---

### ADMIN-5 — Принять оплату (отдельный флоу)

**Назначение:** быстрая фиксация наличных от ученика  
**Шаг 1:** Найти ученика — поиск по имени, выбор из списка  
**Шаг 2:** Выбрать позиции для оплаты — список неоплаченных начислений с чекбоксами, сумма по каждой  
**Шаг 3:** Подтверждение — итоговая сумма, кнопка «Подтвердить получение»  
**После подтверждения:** выбранные позиции отмечаются как оплаченные, экран возвращается к шагу 1 (чтобы быстро принять следующую оплату)

---

### ADMIN-6 — Список должников

**Назначение:** сводная таблица задолженностей  
**Элементы:**
- Итого должников (число) и итоговая сумма долгов по школе
- Фильтр по группе
- Список: имя ученика, группа, неоплаченные позиции (перечисление), сумма долга
- Тап на ученика → профиль ученика на вкладке «Начисления»

---

### ADMIN-7 — Список групп

**Элементы:**
- Кнопка «Создать группу»
- Карточки групп: название, возрастной диапазон, филиал, основной педагог, число учеников
- Тап → профиль группы

---

### ADMIN-8 — Профиль группы

**Секция «Данные»:** название, возраст, филиал, педагог, кнопка «Редактировать»  
**Секция «Ученики»:** список учеников группы с переходом в профиль каждого  
**Секция «Расписание»:** занятия этой группы (день, время, зал, педагог)

---

### ADMIN-9 — Журнал посещаемости

**Навигация:** выбор группы + выбор даты (или конкретного занятия)  
**Элементы:**
- Список учеников с отметками: был / не был
- Для отсутствующих — опциональное поле комментария
- Кнопка «Сохранить»

---

### ADMIN-10 — Расписание (сетка)

**Вид на мобайле:** список занятий по дням (сегодня / эта неделя / следующая неделя)  
**Вид на десктопе:** сетка-календарь (по дням недели и времени)  
**На каждом занятии:** группа, педагог, зал, время  
**Кнопка «Добавить занятие»**  
**Тап на занятие:** редактирование (педагог, время, зал, замена педагога)

---

### ADMIN-11 — Список педагогов

**Элементы:**
- Кнопка «Добавить педагога»
- Карточки: имя, группы, ближайшее занятие
- Тап → профиль педагога (имя, контакт, его занятия)

---

### TEACHER-1 — Расписание

**Элементы:**
- Переключатель: «Все занятия» / «Мои занятия»
- По умолчанию: мои занятия
- Вид: список по дням (сегодня, завтра, эта неделя)
- На каждом занятии: группа, время, зал
- Тап на занятие → переход к отметке посещаемости

---

### TEACHER-2 — Отметка посещаемости

**Навигация к экрану:** из расписания (тап на занятие)  
**Элементы:**
- Название группы, дата и время занятия
- Список учеников группы
- Все по умолчанию отмечены «присутствовал» (зелёный)
- Педагог тапает на ученика чтобы отметить «отсутствовал» (красный)
- При отметке «отсутствовал» — появляется опциональное поле «Причина»
- Кнопка «Сохранить»

---

### STUDENT-1 — Мои начисления

**Элементы:**
- Секция «Не оплачено»: список позиций с суммами, выделено красным / оранжевым. Итоговая сумма долга.
- Секция «Оплачено»: история оплаченных позиций в хронологическом порядке, выделено зелёным
- Простой и чистый вид — родитель должен с одного взгляда понять, сколько нужно принести

---

### STUDENT-2 — Расписание

**Элементы:**
- Расписание группы(й) ученика
- Вид: список по дням недели (повторяющийся)
- День, время, зал

---

### STUDENT-3 — Профиль

**Элементы:**
- ФИО ученика, группа(ы)
- Контактные данные (только просмотр)
- Кнопка «Выйти»

## 5. Общие принципы дизайна

- **Mobile-first**: все экраны проектируются сначала для экрана ~390px, затем адаптируются для десктопа
- **Читаемость**: крупный шрифт для ключевых данных (суммы, имена). Администратор часто работает на ходу
- **Статусы оплаты**: красный = долг, зелёный = оплачено. Эти цвета должны быть мгновенно считываемыми
- **Минимум кликов**: ключевые действия (принять оплату, отметить посещаемость) доступны максимум в 2 клика
- **Пустые состояния**: для каждого списка предусмотреть состояние «пусто» с понятным текстом (например, «Должников нет», «Занятий на сегодня нет»)
- **Подтверждение деструктивных действий**: удаление и деактивация — с диалогом подтверждения
