# ТЗ на дизайн — Итерация 1: Учёт оплат и посещений

## 1. Контекст

Первая итерация приложения для управления танцевальной школой.  
Фокус: учёт платежей, долгов и посещаемости. Расписание как точка входа.

**Язык интерфейса:** русский  
**Платформа:** web, mobile-first (открывается в браузере, без установки)  
**Устройства:** телефон (приоритет) + десктоп  
**Визуальный стиль:** чистый, современный, нейтральный. Спокойная палитра — белый / светло-серый фон, тёмный текст, один сдержанный акцентный цвет на выбор дизайнера. Карточный UI. Минимум декора.

---

## 2. Роли пользователей

| Роль | Описание |
|------|----------|
| **Администратор** | Полный доступ: ученики, платежи, расписание, посещаемость |
| **Педагог** | Видит расписание, список учеников (без финансов), отмечает посещаемость |
| **Ученик / Родитель** | Видит своё расписание, начисления и историю оплат |

---

## 3. Навигация

### Мобильная и десктопная версия — единый подход

Кнопка-гамбургер (три горизонтальные полоски) в **верхнем левом углу** экрана.  
По нажатию открывается боковая панель меню (drawer) поверх контента, с затемнением фона.  
Закрывается тапом на крестик или на затемнённую область за панелью.

**Содержимое панели — Администратор:**
- Расписание
- Ученики
- Должники
- Журнал посещаемости
- Принять оплату
- Выйти

**Содержимое панели — Педагог:**
- Расписание
- Ученики
- Выйти

**Содержимое панели — Ученик:**
- Расписание
- Мои начисления
- Профиль
- Выйти

### Хедер

Присутствует на всех экранах (кроме AUTH-1).  
Содержит: кнопка-гамбургер слева, название текущего экрана по центру.

---

## 4. Стартовая страница

После входа все роли попадают на экран **Расписание**.

---

## 5. Экраны

---

### AUTH-1 — Вход в систему

**Назначение:** единственная страница, доступная без авторизации  
**Элементы:**
- Логотип / название приложения (placeholder)
- Поле: Логин
- Поле: Пароль (скрыт, с кнопкой «показать»)
- Кнопка «Войти»
- Строка ошибки (неверный логин/пароль)

**Примечание:** Регистрация только через администратора — ссылки «Зарегистрироваться» нет.

---

### SCH-1 — Расписание

**Роли:** все  
**Стартовая страница после входа**

**Вид на мобайле:** список занятий по дням — сегодня, завтра, эта неделя  
**Вид на десктопе:** сетка-календарь по дням недели и времени

**На каждом занятии:** группа, педагог, время, зал

**Различия по роли:**
- **Администратор:** видит все занятия школы. Кнопка «Добавить занятие». Тап на занятие → редактирование (время, зал, педагог).
- **Педагог:** видит все занятия школы. Тап на занятие → переход к отметке посещаемости (ATT-2).
- **Ученик:** видит только занятия своей группы. Без возможности редактирования.

**Пустое состояние:** «Занятий на этой неделе нет»

---

### STU-1 — Список учеников

**Роли:** Администратор, Педагог

**Элементы:**
- Строка поиска по имени
- Список учеников: карточка на каждого

**Карточка ученика — Администратор:**
- Имя ученика
- Группа(ы)
- Бейдж «Есть долг» (если есть неоплаченные начисления)

**Карточка ученика — Педагог:**
- Имя ученика
- Группа(ы)
- Без финансовой информации

**Действия:**
- Тап на карточку → профиль ученика (STU-2)
- **Только Admin:** кнопка «Добавить ученика» → форма создания (STU-3)

**Пустое состояние:** «Учеников пока нет»

---

### STU-2 — Профиль ученика

**Роли:** Администратор (полный), Педагог (без финансов)

**Структура: вкладки**

**Вкладка «Данные» — все роли:**
- ФИО ребёнка, дата рождения
- ФИО родителя / ответственного
- Телефон
- Группа(ы)
- Откуда узнали о школе
- **Только Admin:** кнопка «Редактировать» → форма (STU-3)
- **Только Admin:** кнопка «Деактивировать» (с диалогом подтверждения)

**Вкладка «Начисления» — только Admin:**
- Список всех начислений: позиция, сумма, статус (оплачено / не оплачено), дата
- Статус — цветовой маркер: зелёный = оплачено, красный = не оплачено
- На каждой строке: кнопки «Отметить оплату», «Редактировать сумму», «Удалить»
- Кнопка «Добавить начисление»
- Кнопка «Принять оплату» → флоу PAY-1

**Вкладка «Посещаемость» — все роли:**
- Список занятий с отметкой «был» / «не был»
- Фильтр по месяцу

**Пустые состояния:**
- «Начислений нет»
- «Посещаемости нет»

---

### STU-3 — Создание / редактирование ученика

**Роли:** только Администратор

**Поля:**
- ФИО ребёнка (обязательно)
- Дата рождения (обязательно)
- ФИО родителя / ответственного (обязательно)
- Контактный телефон (обязательно)
- Откуда узнали о школе (необязательно)
- Группа(ы) — мультиселект из существующих групп
- Логин (обязательно, только при создании)
- Пароль (обязательно, только при создании)

**Кнопки:** «Сохранить», «Отмена»

---

### PAY-1 — Принять оплату

**Роли:** только Администратор  
**Флоу из 3 шагов**

**Шаг 1 — Найти ученика:**
- Поиск по имени
- Список результатов — тап для выбора

**Шаг 2 — Выбрать позиции:**
- Список неоплаченных начислений ученика с чекбоксами и суммами
- Итоговая сумма обновляется по мере выбора

**Шаг 3 — Подтверждение:**
- Имя ученика, выбранные позиции, итоговая сумма
- Кнопка «Подтвердить получение»

**После подтверждения:** выбранные позиции отмечаются как оплаченные, экран возвращается к Шагу 1

---

### PAY-2 — Список должников

**Роли:** только Администратор

**Элементы:**
- Итого: число должников и общая сумма долгов по школе
- Список: имя ученика, группа, перечисление неоплаченных позиций, сумма долга
- Тап на ученика → профиль ученика (STU-2), вкладка «Начисления»

**Пустое состояние:** «Должников нет»

---

### ATT-1 — Журнал посещаемости

**Роли:** только Администратор

**Навигация к экрану:** из меню  
**Элементы:**
- Выбор группы (дропдаун)
- Выбор даты / занятия
- Список учеников с отметками «был» / «не был»
- Для отсутствующих — опциональное поле комментария
- Кнопка «Сохранить»

---

### ATT-2 — Отметка посещаемости

**Роли:** Педагог (основной путь), Администратор  
**Навигация к экрану:** тап на занятие в расписании (SCH-1)

**Элементы:**
- Название группы, дата и время занятия
- Список учеников группы
- Все по умолчанию отмечены «присутствовал» (зелёный)
- Тап на ученика → переключает на «отсутствовал» (красный)
- При отметке «отсутствовал» — появляется опциональное поле «Причина»
- Кнопка «Сохранить»

---

### MYSTUD-1 — Мои начисления

**Роли:** только Ученик / Родитель

**Элементы:**
- Секция «Не оплачено»: список позиций с суммами, выделено красным / оранжевым, итоговая сумма долга
- Секция «Оплачено»: история оплаченных позиций в хронологическом порядке, выделено зелёным

**Принцип:** простой и чистый вид — родитель с одного взгляда понимает, сколько нужно принести.

**Пустые состояния:**
- «Долгов нет» (в секции «Не оплачено»)
- «Оплат пока нет» (в секции «Оплачено»)

---

### MYSTUD-2 — Профиль

**Роли:** только Ученик / Родитель

**Элементы:**
- ФИО ученика
- Группа(ы)
- Контактные данные (только просмотр)
- Кнопка «Выйти»

---

## 6. Общие принципы дизайна

- **Mobile-first:** все экраны проектируются сначала для ~390px, затем адаптируются для десктопа
- **Хедер:** кнопка-гамбургер слева, название экрана по центру — на всех экранах кроме AUTH-1
- **Читаемость:** крупный шрифт для ключевых данных (суммы, имена). Администратор часто работает на ходу
- **Статусы оплаты:** красный = долг, зелёный = оплачено. Мгновенно считываемые
- **Минимум кликов:** «Принять оплату» и «Отметить посещаемость» доступны максимум в 2 тапа
- **Пустые состояния:** для каждого списка — понятный текст (примеры указаны у каждого экрана)
- **Подтверждение деструктивных действий:** удаление начисления и деактивация ученика — с диалогом подтверждения
