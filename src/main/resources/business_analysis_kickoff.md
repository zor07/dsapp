# Саммари первого созвона — Школа танцев «КИД»

**Дата созвона:** первая встреча (дата не зафиксирована)  
**Участники:** Анзор (разработчик), Дима (владелец/руководитель), Кристина (администратор, жена Димы)  
**Цель встречи:** знакомство с текущими бизнес-процессами школы (as-is)

---

## 1. Общая информация о школе

- **Название:** Танцевальная команда КИД
- **Город:** Белгород

- **Административную работу** ведёт Кристина: запись учеников, расписание, учёт платежей, коммуникация с родителями

## 4. Привлечение новых учеников

### Каналы

- Основной поток — **ВКонтакте**: пост с видео → сообщение в группу или личка
- **Яндекс Директ** → звонок Диме или Кристине
- Иногда — прямой звонок по номеру телефона
- Лиды через VK-форму (приложение «Лиды» в VK)
- Google-форма (редко)

### Сезонность

- Основной набор — **август–сентябрь**
- В разгар сезона (зима–весна) новых почти нет
- Летом сезон закрывается

## 5. Процесс записи нового ученика (as-is)

```
1. Входящий запрос (VK / телефон / звонок)
2. Уточнение данных:
   - Опыт: танцы, гимнастика, спорт, коллектив
   - Откуда узнали о школе
3. Сбор данных ученика:
   - ФИО ребёнка, год рождения
   - ФИО родителя / ответственного
   - Контактный телефон
4. Определение подходящей группы
5. Бесплатное пробное занятие
6. Если нужно ещё одно пробное — разовое занятие (платное)
7. Принятие решения о зачислении
8. Оформление абонемента
9. Добавление в Telegram-группу своей команды
```

**Запись — в группу, не на конкретное занятие.** После зачисления ученик просто приходит по расписанию.

## 6. Расписание

- **Формируется один раз в сентябре** и не меняется до конца учебного года
- Каждая группа имеет своё фиксированное время (например, вторник 18:00–21:00)
- Публично расписание **не публикуется** — сознательная политика (конкуренция, не хотят раскрывать объём работы)
- Расписание доступно только участникам закрытой Telegram-группы (закреплённое сообщение)
- Начало работы — пост в VK и Telegram с расписанием при старте сезона

### Изменения в расписании

- Для **родителей**: фиксированное, меняется крайне редко (праздники)
- Для **педагогов**: плавающее — болезни, концерты, форс-мажоры → замены внутри команды
- Замены педагогов координируются в Telegram-чате педагогов (5 человек)
- Родителей о замене педагога **специально не уведомляют** (по мнению школы — им это не важно)
- Об отменах / переносах пишут в Telegram-группе родителей

## 7. Абонементы и оплата

### Структура абонемента

- Только **месячный абонемент**
- Привязан строго к календарному месяцу (не скользящий)
- Срок оплаты — **до 10-го числа** текущего месяца
- Ученик обязан посещать **все занятия** (командная работа, подготовка к конкурсам)

### Стоимость

Расчёт по часовой ставке внутри абонемента (прогрессивная шкала — чем больше часов, тем дешевле час):

| Кол-во часов в месяце | Ставка |
|----------------------|--------|
| 1–10 ч | ~150 руб/час |
| 11–20 ч | ~140 руб/час |

### Перерасчёт абонемента

| Ситуация | Перерасчёт |
|----------|-----------|
| Пришёл в середине месяца | Да — по фактическим занятиям |
| Болел, есть справка | Да — по пропущенным часам |
| Пропустил по своему желанию | Нет |

### Способ оплаты

- Сейчас — **только наличные** (раньше была возможность перевода на карту, отказались)
- В будущем рассматривают переход на безналичную оплату

### Процесс приёма оплаты (as-is)

```
1. Родитель приносит деньги педагогу или Кристине
2. Родитель пишет сообщение в Telegram-диалог:
   «[ФИО] принесла [сумма] за [абонемент / взнос / форму / ...]»
3. Педагог записывает в бумажную книгу учёта + кладёт деньги
4. Кристина приходит:
   - Открывает книгу учёта
   - Маркером отмечает оплаченное
   - Пересчитывает деньги
   - Забирает
```

**Боль:** очень много наименований платежей, высокая ручная нагрузка на администратора

## 9. Соревнования

### Планирование

- В **августе–сентябре** составляется **календарь событий на полгода**
- На родительском собрании объявляют: «Едем в октябре, ноябре, марте — обязательно»
- Отказ от соревнований = исключение из конкурсного номера (в коллектив ходить можно)

### Группы, участвующие в соревнованиях

| Группа | Кто организует поездку | Что платит родитель |
|--------|----------------------|---------------------|
| Малыши | Родители сами | Взнос + тренерский сбор |
| Дети (конкурсная) | Родители сами | Взнос + тренерский сбор |
| Юниоры / старшая | Школа полностью организует | Взнос + тренерский сбор + поезд + отель |

- Школа ведёт регистрацию участников
- Каждый ребёнок может танцевать разное количество номеров (взнос — за каждый номер)
- Родители, желающие ехать со старшей группой, включаются в общую логистику через школу

## 10. Учёт посещаемости

- Ведётся в **бумажном журнале** (как в обычной школе, ставят «Н»)
- Используется для обоснования перерасчёта абонемента (справка о болезни)
- Любой педагог может отметить посещение
- **Боль:** бумажный журнал, не всегда доступен нужному педагогу

---

## 11. Организационно-правовой контекст

- Школа работает **«в чёрную»** (наличные, без официальных договоров с клиентами)
- **Планы с сентября** — ввести бумажные **договора об оказании платных услуг**:
  - Обязательное посещение
  - Обязательная форма
  - Участие в соревнованиях
  - Условия перерасчёта
- Дима — **ИП на НПД** (налог на профессиональный доход, ставка 6%), работает через чеки
- Понимают необходимость регистрации как **оператора персональных данных** (Роскомнадзор / Минцифры)
- Беспокойство: наличие сумм оплат в приложении и видимость для третьих лиц

## 14. Ключевые боли (зафиксированные на встрече)

1. **Учёт платежей** — основная боль. Много наименований, всё вручную (книга + заметки + Telegram-диалоги)
2. **Учёт посещаемости** — бумажный журнал, неудобен при множестве педагогов
3. **Нет автоматических напоминаний** — родители забывают платить до 10-го числа
4. **Нет единой картины по должникам** — Кристина вынуждена вручную сводить данные
5. **Организация соревнований** — много позиций оплаты по каждому мероприятию, учёт ручной
6. **Расписание педагогов** — координация замен только через мессенджер
